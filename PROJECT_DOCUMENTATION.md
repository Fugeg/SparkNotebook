# GraphRAG 个人知识管理平台 - 项目文档

**项目名称**: 基于 GraphRAG 的个人知识管理与需求决策平台 (AI灵感记事本)  
**开发时间**: 2026.03 -- 至今  
**文档版本**: 1.0  
**最后更新**: 2026-04-12

## 📋 项目概述

### 项目背景
针对传统 RAG 处理碎片化知识时关联弱、逻辑断层的问题，构建基于 **GraphRAG** 与 **MCP** 的智能灵感助手。支持非结构化输入向动态知识图谱的闭环演化，并具备全球开源生态的实时调研能力。

### 核心价值
- **知识关联强化**: 通过 GraphRAG 构建知识图谱，解决碎片化知识关联弱的问题
- **实时开源调研**: 集成 MCP 协议，实时同步全球开源项目趋势
- **智能灵感生成**: 结合本地私有记忆与全球开源生态，生成技术灵感报告

| 设计模式 | 应用 | 优势 |
|---------|------|------|
| **工厂模式** | 统一创建 LLM 提供商 | 解耦实现与使用，易于扩展 |
| **策略模式** | 可插拔故障转移策略 | 灵活配置降级行为 |

**断路器配置**:
- **失败阈值**: 连续 5 次失败触发断路
- **恢复超时**: 30 秒后尝试半开恢复
- **半开测试**: 最多 3 次测试调用
- **恢复阈值**: 连续 2 次成功关闭断路器

### 故障转移策略

#### 1. 立即切换 (IMMEDIATE)
```
请求 ──► StepFun (失败) ──► 立即切换 ──► Qwen-Plus
延迟: ~10-50ms
适用: 高可用性要求的场景
```

#### 2. 渐进切换 (GRADUAL)
```
请求 ──► StepFun (失败) ──► 等待 500ms ──► Qwen-Plus
延迟: ~500ms
适用: 避免瞬时故障误切换
```

#### 3. 预测切换 (PREDICTIVE) [预留]
```
基于历史指标预测故障，提前切换
延迟: ~0ms (预切换)
适用: 智能运维场景
```

- **方式**: nohup 后台运行
- **特点**: 断开 SSH 后继续运行
- **日志**: 输出到 `app.log` 文件
